Berdasarkan hasil proyek yang sudah berhasil dikembangkan sebelumnya, saat ini seiring bertambahnya jumlah pengguna, tim pengembang menemukan beberapa kebutuhan baru yang harus segera diimplementasikan.
Tampilan katalog makanan harus tetap nyaman digunakan pada berbagai ukuran layar Android. Selain itu, halaman Detail Pesanan hanya boleh diakses oleh pengguna yang telah login menggunakan mekanisme autentikasi yang aman. Untuk meningkatkan pengalaman pengguna, aplikasi juga akan ditambahkan fitur kamera sebagai bukti penerimaan pesanan serta maps untuk menampilkan lokasi restoran atau pengiriman.
Sebagai seorang Mobile Developer, Anda diminta untuk mengembangkan dan menganalisis solusi yang sesuai menggunakan React Native.
Fitur utama yang harus dikembangkan, antara lain :

Katalog Makanan
Buat halaman katalog makanan menggunakan komponen <View>, <Text>, dan <Image>.
Terapkan Flexbox agar gambar dan informasi produk tersusun rapi dalam satu baris (row).
Data makanan dapat menggunakan Mock API atau Custom API.

Detail Pesanan
Buat halaman Detail Pesanan yang hanya dapat diakses oleh pengguna yang telah login.
Implementasikan autentikasi menggunakan JWT (JSON Web Token) dan Middleware pada Expo Router untuk melindungi halaman tersebut.

Fitur Camera
Tambahkan fitur kamera untuk mengambil foto sebagai bukti penerimaan pesanan serta tampilkan hasil foto setelah berhasil diambil.

Fitur Maps
Tambahkan halaman yang menampilkan lokasi restoran atau lokasi tujuan menggunakan peta serta tampilkan minimal satu marker pada peta.

Analisis
Jelaskan alasan penggunaan Flexbox dan ukuran proporsional (misalnya flex atau persentase) dalam membangun tampilan yang responsif.
Jelaskan perbedaan Stateful Authentication dan Stateless Authentication (JWT), serta alasan pemilihan JWT pada aplikasi mobile.

API: Mock API atau Custom API
